Tiempo total estimado: 15-20 minutos (solo la primera vez)

PASO 1 — Instalar lo necesario

1.1

Instala Node.js

Node.js es el motor que ejecuta el servidor del juego. Solo se necesita para ejecutar en local (en Render se instala solo).

Si solo quieres jugar ONLINE (sin instalar nada en tu Mac) puedes saltarte este paso. Pero es útil tenerlo para probar en casa.

  1. Abre https://nodejs.org
  2. Pulsa el botón verde grande que dice LTS (la versión estable)
  3. Se descargará un archivo .pkg → ábrelo
  4. Siguiente → Siguiente → Instalar → listo
1.2

Ten GitHub Desktop instalado

Ya lo tienes. Si por alguna razón no lo tienes: desktop.github.com

Ábrelo y asegúrate de haber iniciado sesión con tu cuenta de GitHub.

PASO 2 — Crear cuenta en GitHub

2

Regístrate en GitHub (si no tienes cuenta)

GitHub es donde se guardará el código del juego. Es gratuito.

  1. Ve a https://github.com/signup
  2. Introduce email, contraseña, nombre de usuario
  3. Confirma el email que te envíen

Si ya tienes cuenta, simplemente asegúrate de estar logueado en GitHub Desktop.

No necesitas crear ningún repositorio manualmente. GitHub Desktop lo hace por ti en el paso 3.

PASO 3 — Subir el juego a GitHub

Usaremos GitHub Desktop (la app visual). Si no la tienes: desktop.github.com
3.1

Descomprime el ZIP en tu Escritorio

Haz doble clic en el ZIP descargado. Debería quedarte una carpeta llamada once-upon-a-time-lan.

3.2

Abre GitHub Desktop e inicia sesión

  1. Abre GitHub Desktop
  2. Si es tu primera vez, te pedirá iniciar sesión con tu cuenta de GitHub
  3. Autoriza el acceso
3.3

Añade la carpeta del juego

  1. En GitHub Desktop, ve al menú: File → Add Local Repository...
  2. Navega hasta la carpeta once-upon-a-time-lan y selecciónala
  3. Te dirá "This directory does not appear to be a Git repository"
  4. Pulsa el enlace azul "create a repository" que aparece en ese mismo mensaje
Aparece un diálogo para crear repositorio.
Name: se rellena solo (once-upon-a-time-lan)
Local Path: ya apunta a tu carpeta
Deja todo lo demás como está y pulsa "Create Repository"
3.4

Haz el primer commit

GitHub Desktop ahora muestra todos los archivos del juego en la lista de cambios (columna izquierda).

  1. Abajo a la izquierda, en el campo "Summary", escribe: juego completo
  2. Pulsa el botón azul "Commit to main"
Verás que la lista de archivos se vacía. Eso es bueno — significa que todo está guardado.
3.5

Súbelo a GitHub

  1. Ahora arriba aparece un botón: "Publish repository"
  2. Púlsalo
  3. Aparece un diálogo:
    • Name: once-upon-a-time (puedes cambiarlo)
    • "Keep this code private": DESMARCA esta casilla (necesitamos que sea público para que Render lo vea gratis)
  4. Pulsa "Publish Repository"
Si la casilla "Keep this code private" está marcada, Render no podrá acceder en el plan gratuito. Desmárcala.
¡Listo! Tu código está en GitHub. Puedes verificarlo visitando https://github.com/TU_USUARIO/once-upon-a-time en el navegador — deberías ver todos los archivos.

🔧 Si en el futuro quieres actualizar el juego:

  1. Reemplaza los archivos en la carpeta con la nueva versión
  2. Abre GitHub Desktop — verás los cambios detectados automáticamente
  3. Escribe algo en "Summary" (ej: actualización)
  4. Pulsa "Commit to main"
  5. Pulsa "Push origin" (arriba)
  6. Render redespliega automáticamente en 2-3 minutos

PASO 4 — Despliega en Render (la magia)

4.1

Crea cuenta en Render

  1. Ve a https://render.com
  2. Pulsa Get Started for Free
  3. Elige Sign up with GitHub (lo más fácil — así ya se conectan)
  4. Autoriza el acceso cuando te lo pida
4.2

Crea un Web Service nuevo

  1. En el Dashboard de Render, pulsa New + (arriba a la derecha)
  2. Selecciona Web Service
  3. Pulsa Build and deploy from a Git repository → Next
  4. Busca tu repositorio once-upon-a-time en la lista y pulsa Connect
Si no aparece tu repo, pulsa "Configure account" y dale permisos a Render para acceder a tus repositorios de GitHub.
4.3

Configura el servicio

Rellena estos campos exactamente así:

Name once-upon-a-time (o el nombre que quieras)
Region Frankfurt (EU Central) ← el más cercano a España
Branch main
Runtime Node
Build Command npm install
Start Command npm start
Instance Type Free ← MUY IMPORTANTE, selecciona Free
4.4

¡Despliega!

  1. Baja del todo y pulsa Create Web Service
  2. Render empezará a construir tu app. Verás un log en pantalla con texto verde/blanco.
  3. Espera 3-5 minutos. Cuando veas "⚜ Once Upon a Time — puerto XXXX" al final del log → ¡funciona!
Arriba del todo, Render te muestra una URL tipo:

https://once-upon-a-time-xxxx.onrender.com

Esa es TU dirección. Cópiala.
¡HECHO! Tu juego está online. Esa URL funciona desde cualquier dispositivo con internet, en cualquier parte del mundo.

PASO 5 — Jugar con los alumnos

5.1

Antes de clase

  1. Abre la URL en tu navegador para "despertar" el servidor (tarda ~30 segundos la primera vez)
  2. Crea una sala
  3. Anota el código de sala
5.2

En clase

  1. Escribe en la pizarra:
    https://once-upon-a-time-xxxx.onrender.com
    CÓDIGO: DRAGON427
  2. Los alumnos abren la URL en el navegador de su móvil o portátil
  3. Ponen su nombre y el código de sala
  4. Cuando estén todos, pulsa START
No necesitan instalar nada. Solo abrir una dirección en el navegador. Funciona en iPhone, Android, portátil, tablet... cualquier cosa con internet.

🎉 ¡Y ya está!

Tu juego está online permanentemente.
Solo necesitas la URL y ya puedes jugar cuando quieras.

Preguntas frecuentes

💤 "Tarda mucho en cargar la primera vez"

Normal. El plan gratuito de Render "duerme" el servidor tras 15 minutos sin uso. La primera conexión tarda ~30 segundos en despertar. Solución: abre la URL 2-3 minutos antes de que lleguen los alumnos.

📱 "¿Necesitan los alumnos alguna app?"

No. Solo el navegador (Chrome, Safari, Firefox...). Abren la URL y listo.

🔄 "Un alumno se ha desconectado"

Que vuelva a entrar con el mismo nombre y el mismo código de sala. El servidor le devuelve su mano de cartas automáticamente.

🏠 "¿Puedo también jugar en local sin internet?"

Sí. Los archivos INSTALAR.bat / .command y JUGAR.bat / .command siguen funcionando para jugar en LAN.

🔧 "¿Cómo actualizo el juego si hay cambios?"

Si en el futuro recibes una versión nueva del ZIP:

  1. Reemplaza los archivos en la carpeta con los nuevos
  2. Abre GitHub Desktop — detecta los cambios automáticamente
  3. Escribe algo en "Summary" (ej: actualización)
  4. Pulsa "Commit to main"
  5. Pulsa "Push origin" (arriba)

Render redespliega automáticamente en 2-3 minutos.

💰 "¿Me van a cobrar algo?"

No. Tanto GitHub como Render tienen plan gratuito. Con el plan free de Render tienes 750 horas al mes, que es más que suficiente para uso escolar. Ni siquiera necesitas tarjeta de crédito.

🚨 "Me da error al subir por los archivos de música"

Los MP3 son grandes (~7MB cada uno). GitHub acepta hasta 100MB por archivo, así que debería funcionar. Si da error:

  1. Abre el archivo .gitignore de la carpeta con un editor de texto
  2. Añade la línea *.mp3 y guarda
  3. En GitHub Desktop: commit + push de nuevo

El juego funcionará igual, solo sin las pistas de música opcionales.
